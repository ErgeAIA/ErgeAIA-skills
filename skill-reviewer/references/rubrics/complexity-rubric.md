# 复杂度分档标准

## 六维触发条件

| 维度       | 触发阈值                                                   |
| ---------- | ---------------------------------------------------------- |
| 阶段性产物 | ≥3 类（文档 / 模板 / 脚本 / 数据 中至少三类）              |
| 执行链阶段 | ≥3 个明显阶段                                              |
| 运行时依赖 | ≥1 个外部二进制 / API / 搜索                               |
| 扩展面     | 存在时变数据或规则演进                                     |
| 入口数量   | ≥2 个入口或包含定时触发                                    |
| 脚本生态   | 有 `scripts/` 目录且使用 PEP 723 / `npm:` 等自包含依赖声明 |

## 档位映射

- 0-1 项触发：轻量 → 一篇文档足够，不强制拆分
- 2-3 项触发：中等 → 建议拆出 `references/`
- 4 项触发：中等偏复杂 → 建议拆出 `scripts/` 与 workflow 文件
- 5-6 项触发：复杂 → 完整四层结构 + 校验脚本 + 测试集

## 渐进式披露成熟度（额外维度，不计入档位）

观察该 Skill 是否：
- 主文档 ≤500 行 / 5000 Token
- `references/` 文件均有显式"何时加载"触发条件
- `assets/` 模板按需而非预加载

三项全满足 → 成熟；缺一 → 有改进空间。

## 不同档位的差异化处理

- 轻量 Skill：跳过 W4，简化 W5 至单优先级
- 中等及以上：全流程执行
- 复杂 Skill：W3 必须包含执行轨迹分析（如可获得）
