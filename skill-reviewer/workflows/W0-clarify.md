# W0 澄清与目标对齐

## 目标
在进入复杂度判定与问题扫描之前，先确认用户的真实诉求与评审范围，避免"用户想要合规校验，却收到了八段评审"或反之。

## 触发条件
- 用户首次提交 Skill 仓库 / 文件夹 / SKILL.md 节选
- 用户语义模糊（如「帮我看看这个 Skill」「这个能用吗」）
- 用户未声明评审深度（快速 / 完整 / 仅合规）

## 必问清单（最多 3 项，缺什么问什么）

1. **评审目标**
   - 是想做**合规校验**（PASS/FAIL）还是**深度评审**（八段报告）？
   - 还是两个都要（先合规、再评审）？

2. **评审范围**
   - 是评审**整个仓库**还是**单个 SKILL.md**？
   - 是否包含 `scripts/`、`references/`、`assets/` 等子目录？

3. **使用场景**
   - 这个 Skill 是**自用**还是**对外发布**？
   - 是否已有 Agent 执行轨迹 (Trace) 可一并提交？

## 跳过条件
- 用户已明确说出「校验」/「validate」→ 直接进 V0
- 用户已明确说出「评审」/「review」+ 提交完整仓库 → 直接进 W1
- 用户已附带 Trace → 不必再问，进 W1 并标记 Trace 已就绪

## 输出格式
若需要澄清，仅输出问题，不输出任何评审内容；若已明确，输出一句话：
> 已收到 Skill 仓库，进入<合规校验 / 复杂度判定>流程。
